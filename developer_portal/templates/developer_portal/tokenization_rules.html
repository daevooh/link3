About Tokenization Rules

Tokenization rules define how users earn tokens for performing different actions in your application.

Rule Components
  • Action Type: The type of user action (login, like, comment, etc.)
  • Base Amount: The base number of tokens awarded for the action
  • Cooldown: Minimum time between rewards for the same action
  • Multiplier: Optional factor to multiply the base amount
  • One-time: If enabled, the user can only earn tokens for this action once
Tip: Customize your tokenization strategy to incentivize valuable user behaviors and engagement.
